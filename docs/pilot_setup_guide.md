# Pilot Customer Setup Guide

This document is for onboarding specialists deploying NexusPOS AI at a new pilot location (e.g., Darjeeling Tea Shop, Cafe, or Restaurant).

---

## 1. Hardware Verification
Before arriving on-site, confirm the following hardware is available and operational:
*   [ ] **POS Terminal**: Windows/Mac PC or high-end tablet (iPad Pro / Galaxy Tab).
*   [ ] **Kitchen Tablet**: iPad or Android tablet (at least 10 inches) mounted in a protective, splash-proof casing.
*   [ ] **Receipt Printer**: Thermal 80mm printer (USB, Ethernet, or Bluetooth).
*   [ ] **Network**: Dedicated business Wi-Fi (Do not use public guest networks).

## 2. Software Initialization
1.  Navigate to the designated NexusPOS URL for this tenant (e.g., `teashop.nexuspos.ai`).
2.  Log in using the provided `Owner` credentials.
3.  Open **Settings > Business Settings** and enter the client's actual legal data to ensure initial compliance.
4.  Pair the thermal printer with the POS terminal OS (Windows/iOS) and run a test print from Notepad to ensure driver compatibility.

## 3. Data Migration & Menu Build
If the client is transitioning from an old POS:
1.  Request an Excel/CSV export of their old menu.
2.  Manually create the Categories in NexusPOS.
3.  Manually add the top 20 highest-selling items first to ensure they can trade immediately.
4.  *(Bulk CSV import is planned for Phase 2).*

## 4. On-Site Staff Training
Conduct a 30-minute interactive session:
1.  **Cashiers**: Have them process a fake order with split payments. Show them how to void an item.
2.  **Kitchen**: Have them tap the tablet to move the fake order from PENDING to SERVED. Ensure they hear the audio chime.

## 5. Hypercare Handover
*   Leave a printed copy of the **Quick Start Guide** by the till.
*   Provide the manager with the direct phone number for IT support during the first 48 hours of trading.
